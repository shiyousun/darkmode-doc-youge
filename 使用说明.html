📖 darkmode-doc-youge 使用说明

把 Word / PDF / Excel / 图片一键换成「真护眼」黑底文档。
比纯黑底白字更柔和，眼睛长时间看资料不再累。

适合：

  • 🧑‍🎓 给孩子做电子试卷复习、晚上看作业的家长
  • 📚 长时间阅读 PDF 资料、电子书的上班族
  • 👨‍💻 写代码累了想看暖色调文档的程序员
  • 📷 经常拍纸质资料、网页截图的学习党

⚡ 5 分钟上手

1. 装依赖（首次必装，2-5 分钟）

macOS：

# Python 库
pip3 install --user --break-system-packages PyMuPDF python-docx Pillow openpyxl

# 系统工具（处理 .doc 老格式 + WMF 数学公式必装）
brew install libreoffice libwmf

Ubuntu / Debian：

pip3 install PyMuPDF python-docx Pillow openpyxl
sudo apt install libreoffice libwmf-bin

Windows：建议在 WSL2 里跑（命令同 Ubuntu）。原生 Windows 也可以，但 LibreOffice 路径需要手动配置。

2. 把 skill 放到本地

# 方式一：从 GitHub 克隆
cd ~
git clone https://github.com/shiyousun/darkmode-doc-youge.git
cd darkmode-doc-youge

# 方式二：直接下载 zip 解压到任意目录

3. 跑第一个例子（用你自己的资料）

# 替换成你的目录路径
python3 scripts/darkmode_batch.py "/路径/你的资料夹"

完事——结果在你的资料夹下生成的 dark/ 子目录里，原文件零修改，可以放心试。

🎯 两档模式怎么选？

v2.0 最重要的能力是两档模式，本质区别在「插图要不要也反色」：

模式 命令 适合
默认（不反色插图） python3 scripts/darkmode_batch.py "/路径" 文档里有彩色照片、地图、产品截图、实验图——保留原色更好看
--invert-images（反色插图） python3 scripts/darkmode_batch.py "/路径" --invert-images 文档里以黑白线条为主——比如数学公式、几何图、电路图、流程图——反色才统一好看

真实决策表

文档类型 推荐模式 原因
初中化学/物理 课本 默认 实验照片是彩色的，反色后会发绿/发紫，难看
初中地理/历史 教材 默认 地图、文物照片必须保持原色才能看
商业 PPT、产品介绍 默认 品牌图标、产品截图反色就废了
食谱、旅游攻略 默认 食物照片必须原色
数学试卷、几何题集 --invert-images 公式、几何图都是黑白线条，反色后跟暗色背景融为一体
物理试卷、电路图 --invert-images 电路图都是黑白，反色后变成暖米杏色，视觉统一
法律/医学教材（黑白示意图） --invert-images 示意图反色后更护眼
学术论文（黑白图表） --invert-images 同上

不确定？ 先用默认模式——保留插图原色总是安全的，大不了发现公式还是白底再加 --invert-images 重跑一次。

📋 命令速查

批处理整个目录（最常用 ⭐）

# 默认：仅改背景+文字，不动插图
python3 scripts/darkmode_batch.py "/路径"

# 反色全部：含数学公式、几何图、所有插图
python3 scripts/darkmode_batch.py "/路径" --invert-images

# 换主题
python3 scripts/darkmode_batch.py "/路径" --theme paper      # 仿羊皮纸 sepia
python3 scripts/darkmode_batch.py "/路径" --theme gray       # 暖灰，不那么黄
python3 scripts/darkmode_batch.py "/路径" --theme pure       # 旧版纯黑白（最刺眼，仅兼容用）

# 只处理当前目录不递归
python3 scripts/darkmode_batch.py "/路径" --no-recursive

# PDF 调高清晰度（默认 200dpi，调 300 更清晰但更慢更大）
python3 scripts/darkmode_batch.py "/路径" --dpi 300

# 自定义输出位置（默认在源目录下建 dark/）
python3 scripts/darkmode_batch.py "/路径" --out "/自定义输出位置"

单文件转换

# PDF（PDF 始终整页栅格化反色，无图文区分能力）
python3 scripts/darkmode_pdf.py 输入.pdf [输出.pdf] [--dpi 200] [--theme warm]

# Word（默认保留插图原色）
python3 scripts/darkmode_docx.py 输入.docx [输出.docx] [--theme warm]

# Word（反色含 WMF 公式）
python3 scripts/darkmode_docx.py 输入.docx [输出.docx] --invert-images [--theme warm]

# Excel
python3 scripts/darkmode_xlsx.py 输入.xlsx [输出.xlsx] [--theme warm]

# 单图（jpg/png/jpeg/bmp/gif/tiff/webp，自动 EXIF 旋转校正）
python3 scripts/darkmode_image.py 输入.jpg [输出.jpg] [--theme warm]

🎨 4 套主题怎么选？

主题 背景色 文字色 对比度 视觉感受 适合
warm（默认 ⭐） #1F1F1F 暖深灰 #D4C9A8 暖米杏 ~11:1 类 Kindle Paperwhite Dark 大多数情况，最护眼
paper #1B1A17 烤纸黑 #D8C9A0 旧纸黄 ~12:1 仿羊皮纸 sepia 暗色 读小说、电子书
gray #262626 暖炭灰 #D4D4D4 暖灰白 ~12:1 偏冷淡的护眼方案 觉得 warm 太黄
pure #000000 纯黑 #FFFFFF 纯白 21:1 纯黑白（最刺眼） 旧版兼容，不推荐

为什么不推荐纯黑白？
纯白 + 纯黑对比度 21:1 是公认最伤眼的组合。Apple Dark Mode、VS Code Dark+、GitHub Dark、Kindle 暗色阅读全部不用纯白纯黑——暖色系（偏黄红）能降低蓝光刺激，长时间阅读最舒适。

❓ 常见问题 FAQ

Q1：处理完发现 dark/ 里的图标显示乱码 / 打不开？

A：极少发生，通常是源文件本身格式损坏。
- 检查源文件能不能正常用 Word/Acrobat 打开
- 用 --no-recursive 单独跑这个文件看错误日志

Q2：数学公式还是白底没反色？

A：两个原因可能：
1. 没加 --invert-images 参数 → 默认模式确实不动插图，加上参数重跑
2. 加了参数但漏反色 → 检查 wmf2gd 是否装了：which wmf2gd，没装就 brew install libwmf（macOS）或 sudo apt install libwmf-bin（Linux）

Q3：处理 .doc 老格式特别慢？

A：100+ 个 .doc 用 v2.0 的批量预转换是 28 秒 / 133 个。如果你装的是 v1.0 旧版，请升级到 v2.0：

cd ~/darkmode-doc-youge
git pull

Q4：处理完 docx 文件变大了？

A：正常。
- 默认模式：体积基本不变（甚至小一点）
- --invert-images 模式：因为 WMF 矢量公式被转成 PNG 位图，体积会变大 2-5 倍。这是为了保证公式正确反色的必要代价。
- PDF 处理后体积通常变大（200dpi 整页栅格化图比原 PDF 的文字流大）。可以加 --dpi 150 降清晰度换体积。

Q5：处理完原文件还在吗？

A：在。绝对不动原文件——所有输出统一加 _dark 后缀放到 dark/ 子目录。可以放心试。

Q6：可以中途取消吗？

A：可以，Ctrl+C 即可。已经处理完的文件保留在 dark/，没处理完的下次重跑会从头开始（不会断点续传，但跳过已处理的不会重复做）。

Q7：Mac 第一次跑提示"无法验证开发者"？

A：通常是 LibreOffice 安装时的提示。
- 系统设置 → 隐私与安全性 → 滚到底找到"已阻止 LibreOffice" → 点"仍要打开"
- 或者命令行：sudo xattr -rd com.apple.quarantine /Applications/LibreOffice.app

Q8：能用在 Windows 上吗？

A：能，但建议用 WSL2（Win10/11 自带的 Linux 子系统）。原生 Windows 也可以，但需要：
- 安装 Python 3.10+
- 安装 LibreOffice（手动加到 PATH）
- 安装 libwmf（找 Windows 版二进制，比较麻烦）

Q9：要不要联网？

A：不要。所有处理都在本地完成，零联网、零上传、零隐私泄露。试卷/合同/简历这种敏感文件可以放心处理。

Q10：能改成更深 / 更浅的颜色吗？

A：可以。修改 scripts/themes.py 里的 THEMES 字典，加自己的主题：

"my_theme": {
    "name": "我的主题",
    "bg": "2A2520",        # 背景色 hex（无 #）
    "fg": "E0D5B5",        # 文字色 hex
    "border": "6B5E4C",    # 边框色 hex
    "bg_rgb": (42, 37, 32),
    "fg_rgb": (224, 213, 181),
},

然后用 --theme my_theme 调用。

🛠 故障排除

错误：ModuleNotFoundError: No module named 'fitz'

原因：PyMuPDF 没装。
解决：pip3 install --user --break-system-packages PyMuPDF

错误：缺少 LibreOffice (soffice)，无法处理 .doc

原因：处理 .doc 老格式必须 LibreOffice，但没装。
解决：
- macOS：brew install libreoffice
- Linux：sudo apt install libreoffice
- 装完后命令行能跑通：soffice --version

错误：处理完 docx 里 ! 图片反色失败 (x-wmf): cannot find loader for this WMF file

原因：开了 --invert-images 但没装 wmf2gd。
解决：
- macOS：brew install libwmf
- Linux：sudo apt install libwmf-bin
- 然后重跑

错误：处理大目录时某些文件失败

正常情况。本 skill 是异常隔离设计——单个文件失败不影响其他，最后会列出失败清单。常见失败原因：
- 源文件损坏
- 源文件正被 Word 打开（产生 ~$xxx.docx 锁，自动跳过）
- 文件名含特殊字符（罕见）

把失败的文件单独跑一次看具体错误。

警告：getdata is deprecated

忽略。Pillow 14（2027 年）才会移除，目前不影响功能。
